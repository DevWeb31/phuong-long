# Security Policy

## Reporting a Vulnerability

Si vous découvrez une vulnérabilité de sécurité dans ce projet, merci de :

1. **NE PAS** créer d'issue publique
2. Envoyer un email à : **contact@devweb31.fr**
3. Inclure :
   - Description de la vulnérabilité
   - Steps to reproduce
   - Impact potentiel
   - Suggestions de fix si possible

Nous nous engageons à :
- Répondre sous **48 heures**
- Fournir un fix sous **7 jours** pour les vulnérabilités critiques
- Vous créditer (si vous le souhaitez) dans les notes de version

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |

## Security Measures

Ce projet implémente :
- ✅ Row Level Security (RLS) PostgreSQL
- ✅ Input validation (Zod)
- ✅ TypeScript strict mode
- ✅ HTTPS only
- ✅ Security headers (CSP, HSTS, etc.)
- ✅ Rate limiting
- ✅ RGPD compliance
